# Project Constraints & Learnings

- On this Windows Codex workspace, never reuse `.tmp-test` or rely on the user temp directory for local pytest runs after packaging or escalated commands. Create a unique workspace-local directory under `.pytest-runs\<timestamp>`, pass it explicitly with `pytest --basetemp <path>`, and disable the cache with `-p no:cacheprovider`; reused or fallback temp directories can inherit ACLs that produce `WinError 5: Access is denied`.
- Release UI must not contain placeholder navigation pages or hand-maintained short language lists. Every visible tab must provide its documented workflow, and language selectors must be generated from model capability metadata so the UI exposes only languages supported by the active local transcription or translation model.
- Superpowers shell helpers run through managed Git Bash may lack core utilities such as `dirname`, `wc`, and `tr`. For `scripts/review-package`, always pass an explicit workspace-local `.superpowers\sdd\*.diff` `OUTFILE`; if another helper such as `scripts/task-brief` fails on a missing core utility and has no output-path override, create the equivalent workspace-local `.superpowers\sdd\*` artifact directly instead of retrying the same shell environment.
- Compute-device availability preflight applies only to production queues; demo queues must always construct their normal demo worker regardless of saved CUDA/fallback policy.
- When extending inference-adapter constructors, preserve every existing positional parameter slot; add new optional configuration as keyword-only unless a versioned breaking change is explicitly approved.
